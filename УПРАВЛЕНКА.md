# Управленка (управленческий P&L) — контекст разработки

Документ собирает весь контекст по фиче «Управленка» из рабочего чата, чтобы продолжить разработку на отдельной ветке `upravlenka`.

## 1. Суть и цель
Управленческий учёт по 3 барам / 2 юрлицам. P&L на текущий календарный год, по каждому юрлицу отдельно. Прошлые месяцы — факт, текущий — факт+прогноз, будущие — прогноз. Выручка реальная (Квикресто), затраты — из 1С (подключается позже). Сейчас собираем **ШАБЛОН**: выручка живая, статьи затрат — пустые «—» до подключения 1С.

## 2. Юрлица и точки
- **ХСБ — «Хоп Сити Барс»** = entity `E1` = Шоссе Энтузиастов (location_id 1) + Строгино (location_id 2).
- **ХСК — «Хоп Сити Концепт»** = entity `E2` = Флакон (location_id 3).
- Налоговый режим обоих: **УСН «Доходы минус расходы» 15%** (`mgmt.entity.tax_regime='usn_profit'`, `tax_rate=0.15`). ИНН не заданы (нужны для 1С).

## 3. Конструкция (последний спец пользователя)
- **Две отдельные вкладки** на сайте: одна — ХСБ, другая — ХСК (НЕ два столбца в одной таблице).
- В каждой вкладке — таблица: **строки = статьи P&L, столбцы = месяцы текущего года (2026, Янв…Дек)** + столбец «Год/итого».
- Раскраска ячеек: прошлые месяцы — факт, текущий месяц — факт+прогноз, будущие — прогноз (прогнозные ячейки помечать визуально).

## 4. Статьи (строки P&L)
**Выручка** (из Квикресто):
- Выручка пиво
- Еда
- Безалко
- Прочая выручка
- = Итого выручка

**Затраты** (из 1С, позже):
- ФОТ
- Премии с ФОТ
- Закупки пиво
- Закупки еда
- Закупки безалко
- Прочие закупки
- Арендная плата
- Чистящие и моющие средства
- Расходы на гостей
- = Итого затраты

**Налоги** (УСН 15%)
**Финансовый результат** (= выручка − затраты − налоги)

## 5. Источники данных
- **Выручка** — есть в боевом Supabase (`ywtsigztqdktpljdmhqp`):
  - Тотал по месяцу/локации: `public.revenue_by_group_daily` (исправлен по Строгино, см. ниже).
  - Категорийный микс (включая «безалко»): `public.product_sales_daily` (есть имя товара + grp).
  - ⚠️ `public.sales_daily` НЕ использовать — битая.
  - Прогноз: `mgmt.revenue_forecast` (тотал на локацию на 12 мес) × категорийный микс последних месяцев.
- **Затраты** — из **1С:Бухгалтерия** (аренда в облаке у хостера), через OData. Слой `buh` + edge-функции `sync-1c`/`push-1c`. Пока НЕ подключено — нужен OData-доступ (заявка хостеру + 2 пользователя + ИНН). Инструкция: Obsidian `Бары/1С OData — инструкция подключения.md`.
- **Налоги** — расчёт УСН 15% = (выручка − затраты) × 15%, минимум 1% выручки. Пока затраты=0 — показывать «—» либо считать на неполных данных (помечать).

## 6. Категория «безалко»
Отдельной группы в данных НЕТ — лимонады/чай/безалкогольное лежат в grp `beer`. Выделяется по названию товара (regex): `лимонад|безалк|0%|б/а|морс|сок|вода|чай`. Маппинг категорий:
- **пиво** = grp `beer` И не безалко
- **еда** = grp `food`
- **безалко** = товары по regex (из любой группы)
- **прочая** = grp `snacks` + `other` (и не безалко)

Текущая разбивка 2026 (продакшн-данные, требует подтверждения классификации безалко):

## 8. RPC для дашборда (public, SECURITY DEFINER, grant anon)
Существующие (старая вкладка «Управленка», будут заменены): `mgmt_summary`, `mgmt_pnl_monthly`, `mgmt_revenue_forecast`, `mgmt_pnl_by_entity`, `mgmt_cashflow`.
Для карточек выручки: `bar_compare()` (прошлый месяц те же дни + прошлый год тот же период).

**Нужно для новой конструкции:** RPC, отдающий по юрлицу × месяцу × статье: выручка по категориям (факт+прогноз) + затраты (пока 0/из 1С) + налоги + результат. Запрос для разбивки выручки (Этап 1):
```sql
with cls as (
  select lm.entity_code, to_char(ps.sale_date,'YYYY-MM') ym,
    case
      when ps.product ~* '(лимонад|безалк| 0%|б/а|нонэлк|морс| сок| вода|чай в ассорт)' then 'nonalc'
      when ps.grp='beer' then 'beer' when ps.grp='food' then 'food' else 'other' end cat,
    ps.revenue
  from public.product_sales_daily ps
  join mgmt.location_meta lm on lm.location_id=ps.location_id
  where ps.sale_date >= '2026-01-01')
select entity_code, ym,
  round(sum(revenue) filter (where cat='beer')) пиво,
  round(sum(revenue) filter (where cat='food')) еда,
  round(sum(revenue) filter (where cat='nonalc')) безалко,
  round(sum(revenue) filter (where cat='other')) прочая
from cls group by entity_code, ym order by entity_code, ym;
```
NB: для исправленных тоталов брать тотал из `revenue_by_group_daily`, а доли категорий — из `product_sales_daily` (иначе Строгино за свежие дни занижается, см. п.10).

## 9. Прогноз для сетки
- Текущий месяц = факт MTD (revenue_by_group_daily) + прогноз остатка месяца.
- Будущие месяцы = `mgmt.revenue_forecast` (тотал на локацию) → агрегировать в юрлицо → разложить по категориям по миксу последних 3 мес.
- Затраты будущих месяцев: пока неясно как прогнозировать (из 1С только факт) — открытый вопрос.
